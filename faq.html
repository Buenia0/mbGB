FAQs

General

What is libmbGB?

libmbGB is a next-gen Game Boy (and Game Boy Color) emulator. The project originally started in October 2018, when I was still a high-school student. However, noting the gross inaccuracy of this older version, the project was rewritten from the ground-up in Summer 2019, with a focus on cycle-accuracy.

Further along in development, libmbGB gained support for the Game Boy Printer and a few other basic addons. Other goals include support for a vast array of Game Boy peripherals, both obscure and more well-known, as well as a good workflow for tool-assisted runners, and a modern feature set for emulators that older emulators may not support.

What are the system requirements for libmbGB?

The core libmbGB engine and its console frontend should run on mostly any computer from the last 5 years or so, including the following OSes:

  • Windows (7 or newer)
  • mac OS (not tested)
  • Linux (only tested on Ubuntu)
  • BSD (not tested)

If you wish to run libmbGB on other systems that RetroArch runs on, please us the libretro core instead. Please note that the libretro core may not compile succesfully on every platform, especially those without support for C++17.

What does the mb stand for?

The mb stands for the name of my dad, Michael B., who, growing up, was also in the tech field, just like I am now. So, in order to honor him, I chose the phrase "mb" as a universal name for the family of emulators I planned to create, of which libmbGB is just one member.

How do I file a bug?

The best way is to sign up for an account on GitHub and post on the issue tracker. Please include the version you're using, what OS you're running on, and the names of games affected.

What is the license for libmbGB?

libmbGB is licensed under the GNU GPL v3.0, and the code can be found on GitHub.

Features

How good is compatibility?

Overall, compatibility is good. Most games work but there are still plenty of bugs. Please file a bug if you find a game that suffers from issues and/or doesn't boot.

Current bugs can be found on GitHub's issue tracker for this project.

What peripherals and cartridge types does libmbGB support?

The following Game Boy addons/special cartridge types are supported:

  • Rumble
  • Tilt (MBC7)
  • Real-time clock
  • Game Boy Printer (currently being rewritten)
  • Mobile Adapter GB (currently being rewritten)
  • Link Cable (currently being rewritten)
  • Barcode Boy (currently being rewritten)

The following are not supported at this time:

  • MBC6
  • MMM01
  • TAMA5
  • HuC-1
  • HuC-3
  • Turbo File GB
  • GBC Infared
  • Barcode Taisen Bardigun
  • Pocket Sonar
  • Sewing machines (such as Singer Izek 1500, etc.)
  • Power Antenna/Bug Sensor
  • DMG-07 (Four Player Adapter)
  • Full Changer (for Zok Zok Heroes)

Troubleshooting

On loading Mortal Kombat 1/2 or Bomberman Collection I have issues with the game booting/making it past the menus!

This is a known issue with the libmbGB core. If playing these games is your utmost concern, you may want to enable MBC1 multicart emulation in libmbGB.

Hint: This can be accomplished in the SDL2 frontend by passing the "--mbc1m" argument to the program.
